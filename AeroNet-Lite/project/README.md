## What is AeroNet Lite?

AeroNet Lite simulates an autonomous drone delivery system over a 10×10 city grid. It integrates five AI modules that work together in a 20-step simulation: validating the city layout, selecting a drone fleet, planning delivery routes, handling mid-flight disruptions, forecasting demand, and detecting flight anomalies. Results are visualised through an interactive Streamlit dashboard.

## Simulation Overview

The simulation runs for 20 steps. Here is what happens at each stage:

| Steps | Event |
|---|---|
| 1–3 | Grid initialised, CSP layout validated, fleet selected by GA |
| 4–6 | Deliveries generated, routes planned with A*, drones dispatched |
| 7–10 | Drones fly along planned routes |
| 11 | No-fly zone activated on an active route — affected drones rerouted via A* |
| 12–14 | Rerouting completes, drones continue flying |
| 15–17 | Demand forecast loaded, remaining deliveries assigned |
| 18 | Anomaly injected into a random drone |
| 19 | Emergency hub return if battery critical, otherwise drones continue |
| 20 | Final summary: completed, delayed, and failed deliveries printed |

## CSP Layout Rules

| Rule | Constraint |
|---|---|
| R1 — Industrial Safety | Industrial cells cannot be adjacent to Schools or Hospitals |
| R2 — Residential Coverage | Every Residential cell must be within 3 Manhattan cells of a Hub |
| R3 — Hub Charging | Every Hub must have a Charging Pad within 2 cells |
| R4 — Medical Access | At least one Hospital must have a Medical Pickup within 1 cell |

## ML Pipeline

**Demand Forecasting** — trained on Bike Sharing Demand dataset (10,886 records).
Features: season, holiday, workingday, weather, temp, humidity, windspeed, hour.
Models: Linear Regression and Random Forest Regressor evaluated on MAE and RMSE.

**Anomaly Detection** — trained on 950-record synthetic drone telemetry dataset.
Classes: Normal, Battery Anomaly, Route Anomaly, Sensor Spike.
Features: battery_drop, speed, route_deviation, altitude_change, speed_change.
Models: Decision Tree and Random Forest Classifier evaluated on accuracy and confusion matrix.

## Notes

- All modules use a fixed `random.seed(42)` for reproducibility.
- The simulation generates 5–10 deliveries depending on configuration. Change `count` in `run_simulation()` inside `delivery_simulator.py`.
- The no-fly zone is always placed on an active drone route to guarantee a rerouting event occurs.
- Anomaly injection happens at step 18 on a random active drone. If battery drops below 20% the drone returns to its hub automatically.
